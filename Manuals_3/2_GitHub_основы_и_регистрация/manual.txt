- Общий рассказ о Git и общее понимание зачем все это нужно.


Мы дошли до момента знакомства с очень интересной и важной темой - система контроля версий.
Таких систем существует большое количество. Мы будем работать с самой популярной из них - Git.

Система контроля версий (СКВ) - средство, помогающее проще отслеживать ход работы над каким-либо проектом, смотреть
на изменение файлов и возвращаться на предыдущие версии.
Таким средством пользуются все современные компании и IT-корпорации.

Самая популярная СКВ - Git. Она была создана 7 апреля 2005 года программистом Линусом Торвальдсом для упрощения
разработки гео проекта - ядра операционной системы Linux. С тех пор её активно дорабатывают и улучшают.

Что такое репозиторий - хранилище - место, где хранятся и поддерживаются какие-либо данные, чаще всего одного проекта.
Для каждого своего проекта можно создать репозиторий, за которым Git будет следить. Аналогом репозитория можно назвать
проект в pyCharm.

СКВ Git имеет древовидную конструкцию. Остновным стволом является ветка master. Чаще всего она используется для
основного проекта. Git позволяет создать также и отдельную(ые) ветку(и), чтобы в ней(них) можно было тестировать и
писать новый код, не влияющий на основной проект. Ветки независимы друг от друга, но при желании их можно объединять
(merge - слияние), даже если между ними есть разница в коде.

Коммит - загрузка нового кода в текущую ветку. Для коммита записывается временная метка и комментарий, в котором
можно указать назначение коммита. Это запись изменений в проект.

Бывает такое, что внесенные в код измененния больше тебя не устраивают, необходимо вернуть старый код. Git позволяет
выбрать нужный коммит и вернуться к нему, обновив весь проект.

Получается, что Git - всего лишь программа, позволяющая управлять файлами.

Установим Git на компьютер: https://git-scm.com/downloads

GitHub - это облачная платформа для хостинга IT-проектов и совместной разработки, а также полноценная социальная
сеть для разработчиков. GitHub позволяет загружать свои проекты на сайт и вести разработку напрямую там. Теперь
консольный Git превращается в визуальный GitHub.
